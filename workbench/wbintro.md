# Introduction

The KCML workbench builds on the existing editing code, but now moves forward. It supports a multi-window environment, with optional windows for variables, watch points, objects, the return stack etc. The menus are augmented by toolbar buttons and in most places a right click will give a context specific menu taking you where you want to go. There is extensive use of tooltips and context menus. By using separate windows and using toolbars, tree controls and other modern features the aim is to give the programmer as much help possible right away. Thus the object browser is a tree control with forms, controls, properties, methods and events all in a clear hieirachy next to your code. You can even drag items off the object browser and place them where you want directly in the editor. The display of programs no longer uses fixed color and attribute sequences allowing much more efficient ways of displaying the program.
