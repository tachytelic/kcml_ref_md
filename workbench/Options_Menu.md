## Config Menu

These menu options allow the configuration of the development environment.

<img src="bitmaps/configmenu.png" data-align="BASELINE" data-border="0" alt="configure menu" />

### \$OPTIONS ...

Not yet available

### Collapse REM statements

This function, also available as a keyboard toggle with CTRL-R, will consolidate multi line REM statments into one line thus exposing more of the code.

### Color options ...

Displays a [dialog](coloropts.htm) that allows the default colors and fonts to be modified. Modifications are saved into the registry under the current user.

### Autosave options

Displays a [dialog](autosave.htm) that allows the autosave filename and the delay between autosaves to be modified.

### Cursor

This brings up a secondary menu allowing the editor cursor shape to be chosen from **Bar**, **Line** or **Block**. The default is for a bar cursor.

### Edit scene

This brings up a submenu allowing you to specify the [window scene](scene.htm) to be used when switching into the editor from debug mode. The default is **No Change**, that is to stay in the same scene.

<img src="bitmaps/scenemenu.png" data-border="0" alt="Choose edit/debug scene menu" />

### Debug scene

This brings up a submenu (identical to the edit scene one above) allowing you to specify the [window scene](scene.htm) to be used when switching into the debugger from the editor.
