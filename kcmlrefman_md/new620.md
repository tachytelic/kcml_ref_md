# What's New in KCML 6.20

> Summary of major new features introduced in KCML 6.20.

## Records

- `LOCAL DEFRECORD` now supported inside subroutines.
- Simplified notation for nested field access: `FLD(FLD(a$.b$).c)` can be written as `FLD(a$.b$.c)`.
- Secondary records via SYM: `FLD(a$.b.c)` accesses field `c` of a record pointed to by the `SYM` in field `b` of `a`.
- Arrays of secondary records with subscript notation: `FLD(a$.b<n>.c)`.

## Libraries (formerly modules)

- `MODULE` deprecated in favour of `LIBRARY`. `MODULE ADD` → `LIBRARY ADD`. Programs are updated automatically on save.
- `PUBLIC` and `PRIVATE` keywords: use `DIM`, `DEFSUB`, `DEFRECORD` with these qualifiers to control symbol visibility. Default is `PUBLIC`.
- `PUBLIC DIM` replaces the KCML 6.10 `COM` convention for library variables (still accepted in 6.20 but will be removed in the next release).
- A built-in library provides `DEFRECORD` definitions for `$PSTAT`, `$MACHINE`, and common `$DECLARE` definitions. Can be overridden by explicitly loaded libraries.

## Error handling

- **`TRY`/`CATCH`/`THROW`**: Structured exception handling block. Replaces `ERROR DO` and the deprecated `ON ERROR`. Supports nesting and re-throwing. Use `THROW` to raise errors programmatically.

## Forms

- Enhanced forms editor.

## Database

- Enhanced KISAM database support.

## Xerces

- Xerces XML parser integration updated.

## Compatibility changes

- `LIBRARY` is now a keyword (replaces `MODULE`).
- `PRIVATE` and `PUBLIC` are now keywords.
- Runtime field lookup in libraries is the default (`$OPTIONS RUN` byte 49 = `HEX(03)`).
- `KI_FLD` no longer creates field variables — only initialises existing ones.
- `KI_DESCRIBE_COL` returns new `$PACK` text formats.
- Some `$PACK(F)` formats now require `$PACK(E)`.
- `SY8` pack format renamed to `FP`.
- Direct telnet mode (`kcml -l`) removed.
- Loading from platters (`kcml -w`) removed.
