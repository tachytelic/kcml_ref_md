# What's New in KCML 6.10

> Summary of major new features introduced in KCML 6.10.

## Records and fields

- **`DEFRECORD` / `FLD`**: Declare structured records with named fields, eliminating manual byte-offset calculations. KCML enforces field layout and prevents accidental modification.
- Records can be browsed in the Workbench.
- Standard KCML structures documented with records: `$PSTAT`, `LINPUT+`, `KI_INFO`.

## Fields (enhanced)

- New naming scheme for a standardised data-typing system.
- New field types: `DATE`, `TIME`, `HEX`, `BOOL`, `MB` (legacy Y/N string boolean), `TIMESTAMP` (milliseconds).
- `FP` pack type replaces the old `SY8` name.
- `PACK()` function returns the new mnemonic pack format for a field.
- `DIM()` function returns the occurrence number for a field.
- **Compatibility:** `KI_DESCRIBE_COL` now returns new pack codes rather than 2-byte hex strings — applications may need updating.

## Constants

Constants are now enabled by default. Numeric variables starting with `_` and `DEFFN` functions defined as constants are treated accordingly.

## Modules/Libraries

The term "module" is now "library" — `MODULE ADD` should become `LIBRARY ADD` (both still accepted). `PUBLIC DIM` replaces the `COM` convention for library variables.

## XML DOM

Xerces XML DOM parser support added via the dynamic object mechanism.

## SOAP

SOAP client and server support added (see `ObjSoap`, `soapserver`).

## COM enhancements

OCX96 specification fully supported in forms.

## Unicode

UTF-8 string iteration functions: `ULEN8(`, `UNEXT8(`, `UPREV8(`.

## Other changes

- Oracle database backend for KISAM.
- `$TIMESTAMP` returns millisecond timestamp.
- Enhanced forms data-aware controls.
- `PUBLIC` and `PRIVATE` keywords for library symbol visibility.
