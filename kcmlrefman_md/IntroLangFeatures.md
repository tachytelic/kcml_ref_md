# Overview of Language Features

> A summary of KCML's built-in capabilities: maths, strings, matrices, file I/O, GUI forms, OS integration, and security.

## Overview

KCML provides a broad standard library covering the most common needs of business application development. Rather than requiring external libraries for sorting, string searching, database access, or GUI forms, KCML handles these at the language level.

## Maths Functions

Built-in mathematical functions cover:

- Trigonometric: `SIN(`, `COS(`, `TAN(`, `ARCSIN(`, `ARCCOS(`, `ARCTAN(`
- Rounding and truncation: `INT(`, `ABS(`, `RND(`
- Modulus arithmetic
- Random number generation
- Boolean operations on numeric values

## String Functions

**Fields:** KCML's field mechanism allows named sub-regions of a string buffer to be declared once and referenced by name throughout the program. This is equivalent to struct member access in C and makes record-handling code self-documenting.

**Bit operations:** Logical `AND`, `OR`, and `XOR` can be applied to all or part of a string variable, treating each byte as a bit pattern. Binary and decimal additions and subtractions can also be performed on entire strings or substrings.

## Matrix Handling

KCML supports matrices of both numeric and alpha data with up to two dimensions. Built-in statements allow:

- Searching and sorting array contents
- Extending arrays at runtime (`MAT REDIM`)
- Initializing all elements in one statement (`ZER`, `CON`, `ALL(`)
- Standard matrix arithmetic on whole numeric arrays (addition, subtraction, scalar multiply, etc.)

## Forms (GUI)

KCML has native support for GUI forms displayed on a connected Kclient. Forms are designed in the Workbench with a graphical editor. Event handlers written in KCML attach to user interactions (button clicks, field changes, etc.). The server-side KCML program drives the form; the client renders it.

## Distributed Objects

KCML can instantiate and interact with COM and CORBA servers directly, using native syntax for methods and properties. The `$DECLARE` statement provides direct access to Windows and third-party APIs.

## File I/O

KCML provides three levels of file access:

| Method | Description |
|--------|-------------|
| **KISAM** | The native built-in relational database. Fast B-tree indexed access. Supports both SQL queries and direct row-set (ISAM) operations. Preferred for all structured data. |
| **ANSI-style hash I/O** | `OPEN #`, `READ #`, `WRITE #`, `SEEK #`. Works on regular OS files, UNIX devices, and UNIX pipes. Simpler but lower performance than KISAM. |
| **BASIC-2 platter style** | Legacy compatibility with Wang 2200 file layout. Supported but discouraged — it is inefficient and has significant restrictions. An enhanced variant stores files in the native OS filesystem. |

## Device I/O

KCML can address any peripheral connected to the Unix host or Windows PC, including network devices. Both standard input and standard output can be redirected to alternative devices. TCP/IP sockets, serial ports, pipes, and native files are all accessible in a platform-neutral way.

## Native Operating System Access

KCML programs can execute native OS commands and utilities at any time, including during development. The `$DECLARE` statement allows direct calls into Windows DLLs and third-party APIs.

KCML intentionally wraps most OS-specific functionality behind its own portable commands (sockets, serial ports, pipes, native files) so that programs do not become tied to a single platform. Excessive direct OS calls compromise the write-once portability goal.

## Programming Constructs

KCML has a full set of structured programming constructs:

| Construct | Purpose |
|-----------|---------|
| `FOR … NEXT` | Counted iteration with optional `STEP` |
| `WHILE … WEND` | Pre-test loop |
| `REPEAT … UNTIL` | Post-test loop |
| `BREAK` | Exit a loop immediately |
| `CONTINUE` | Skip the rest of the current loop iteration |
| `SELECT CASE` | Multi-way branch on a single expression |
| `DO … ENDDO` | Statement grouping block |

## Security

**Halt key protection:** If the `HALT` key is pressed by a user, KCML displays a password prompt. The correct response is time-limited and generated by the software supplier, so a user cannot capture a password and reuse it later. This is enabled via `$OPTIONS`.

**Scramble protection:** Programs can be scramble-encrypted to prevent copying or modification. A one-shot password mechanism (similar to the Halt key mechanism) allows a vendor to authorize a specific unscramble operation over the phone without giving away the scramble key.

## Data Conversion and Storage

**`CONVERT`:** Converts alpha variables to numeric and vice versa in a single statement.

**`$PACK` / `$UNPACK`:** Pack multiple numeric values into a compact binary representation within an alpha variable. Supports many different storage formats. The reverse operation (`$UNPACK`) extracts the original numeric values. This is the basis for efficient record layouts.

## Notes

- KISAM (the built-in database) is discussed in detail in the File I/O chapters. For new development, prefer KISAM or the ANSI hash I/O methods over the legacy BASIC-2 platter format.
- The `$DECLARE` statement is Windows-specific. Use it sparingly to avoid breaking portability to Unix platforms.
- See also: `$PACK`, `$UNPACK`, `CONVERT`, `FOR`, `WHILE`, `REPEAT`, `SELECT CASE`, `DO`, `BREAK`, `CONTINUE`, `$OPTIONS`, `$DECLARE`
