# KCML Statements and Commands — Introduction

> How to read the KCML language reference pages.

## Page layout conventions

Each instruction page in the reference contains:

### Instruction title

The instruction name and type on the first line:
- **Function**: used in numeric or alphanumeric expressions; has an opening `(` immediately after the name.
- **Statement**: a programmable instruction.
- **Command**: a non-programmable instruction (immediate mode only); the word `command` appears next to the name.
- **Operator**: operates on alpha-operands; the word `operator` appears next to the name.

Instructions usable both in programs and immediate mode have no type label.

### General Form

Describes the syntax using these conventions:

| Convention | Meaning |
|------------|---------|
| Uppercase letters | Literal keyword or instruction text — enter exactly as shown |
| Lowercase words | Information to be supplied by the programmer |
| `{ A`  /  `B }` (braces with vertical list) | One option must be chosen |
| `[...]` (square brackets) | Optional — use any one option or none |
| `...` (ellipsis) | The preceding element may be repeated one or more times |
