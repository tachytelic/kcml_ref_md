# MAT MOVE

> Transfers array elements one-by-one from a source array to a receiver, optionally using a locator array for indirect addressing.

## Syntax

```
MAT MOVE move_array [(x [,y])] [, locator_array] [, n] TO receiver_array [(x [,y])]
```

| Element | Description |
|---------|-------------|
| `move_array` | Source array (alpha or numeric) |
| `(x [,y])` | Optional subfield of each alpha element: `x`=start, `y`=length |
| `locator_array` | Optional: alpha array of 2-byte binary element subscripts (for indirect transfer) |
| `n` | Optional counter: in = max to transfer, out = actual transferred |
| `receiver_array` | Destination; can specify start element e.g. `b(5)` |

## Description

MAT MOVE is much faster than a KCML loop for element-by-element array transfers.

**Direct transfer** (no locator): copies elements sequentially from source to receiver.

**Indirect transfer** (with locator): each 2-byte entry in the locator array specifies a subscript in the source array. Transfer stops when:
- End of source/locator reached
- Binary `HEX(0000)` found in locator
- `n` elements transferred
- Receiver is full

For 2D source arrays, the locator uses two 1-byte subscripts per element (instead of one 2-byte). This limits indirect addressing to arrays ≤65,535 elements.

### Subfield transfer

```kcml
DIM a$(10)16, b$(20)10
MAT MOVE a$()(2,4) TO b$()
```

Copies 4 bytes starting at byte 2 from each element of `a$()` into `b$()` — equivalent to:

```kcml
FOR i = 1 TO DIM(a$(),1)
    b$(i) = STR(a$(i), 2, 4)
NEXT i
```

## Examples

```kcml
REM Simple direct move
MAT MOVE source$() TO dest$()

REM Start at element 5 of destination
MAT MOVE a() TO b(5)

REM With counter
MAT MOVE input$(), count TO output$()
PRINT "Moved: "; count; " elements"
```

## Notes

- Locator arrays are typically generated by `MAT SORT` or built programmatically.
- The 2-byte element limit restricts locator-based indirect moves to arrays with at most 65,535 elements; use a numeric pointer variable for larger arrays.
- Deprecated: mixing alpha and numeric arrays in MAT MOVE (BASIC-2 compatibility) — not documented.
