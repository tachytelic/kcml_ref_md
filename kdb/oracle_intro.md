Oracle

The Kerridge Database API has been extended to permit operation against other SQL databases and the first one to be supported is Oracle (version 8 and above). While both Oracle and the Kerridge Database are relational databases, Oracle exposes data only through a SQL programming language whereas in KCML the underlying ISAM structure can also be addressed directly which permits greater flexibility and offers perfomance advantages. A mapping layer has been added to KCML which translates the standard database API operations into equivalent SQL which is then executed against the Oracle database. This has been done in such as way as to require almost no changes to existing KCML programs in order to migrate KCML tables to Oracle tables.

This implementation permits the conversion of some KCML tables to Oracle format so that they can be shared with other systems. It also allows an otherwise KCML based system to access Oracle tables from another system. Note that Oracle is built on a network architecture so the other database may be on another processor. Two phase commit protocols are not supported so updating both an Oracle database and a KDB database in the same transaction is not possible.
